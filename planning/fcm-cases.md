## 4. 발송 방식 (참고)

| 발송 방식 | 어떤 알림이 해당되는가 |
|---|---|
| **즉시 발송** | 1번 좋아요, 2번 인기글, 3번 게시글 댓글, 5·6번 투표 좋아요/댓글, 8·9번 다짐 응원/댓글, 11번 배지, 12·13번 학생증 인증 결과 |

## 5. 새 알림 추가하기 (기획자 작성 가이드)

### 5.1 새 알림 정의 시 결정해야 할 항목

새 알림을 만들 때 아래 6가지 질문에 답을 정합니다.

1. **언제 발송되는가?** (어떤 사용자 행동 직후 / 어떤 시간에 / 어떤 조건이 충족됐을 때)
2. **누구에게 발송되는가?** (특정 사용자 1명 / 일부 사용자 / 전체 학생)
3. **알림 제목은?** (권장 20자 이내, 짧고 명확하게)
4. **알림 내용은?** (권장 60자 이내, OS 미리보기에 잘리지 않도록)
5. **탭하면 어디로 이동하는가?** (기존 화면 중 하나 / 새 화면 필요)
6. **사용자가 끄고 싶어할까?** (수신 거부 설정 추가 여부)

### 5.2 작성 체크리스트

- [ ] 동일하거나 유사한 알림이 이미 있는지 확인했는가? (중복 방지)
- [ ] 알림 제목이 20자 이내인가?
- [ ] 알림 내용이 60자 이내인가?
- [ ] 변수가 들어간다면 `{변수명}` 형식으로 명시했는가?
- [ ] 탭 시 이동할 화면이 정해졌는가? (없으면 알림센터로 이동)
- [ ] 수신 거부 설정 제공 여부를 정했는가?

### 5.3 새 알림 행 템플릿

아래 행을 1번 마스터 테이블에 복사해 채우세요.

```
| N | (카테고리) | (알림 이름) | (발송 조건) | (알림 제목) | (알림 내용) | (수신자) | (탭 시 이동 화면) |
```

### 5.4 추가 시 영향받는 영역 (개발팀 협의용)

새 알림 추가는 단순한 문구 추가가 아니며, 아래 영역에 변경이 생길 수 있습니다. 기획서 작성 시 개발팀과 사전에 공유합니다.

- 백엔드 — 알림 발송 코드, 알림 종류 enum
- 사용자 알림 설정 — 수신 거부 설정이 새로 필요하면 설정 화면에도 추가
- 프론트 — 알림 종류 enum, 탭 시 이동 분기 (새 화면이면 신규 라우팅 필요)
